# <a name="getting-started-with-power-bi-gateways"></a>เริ่มต้นใช้งานเกตเวย์ Power BI
ยินดีต้อนรับสู่คำแนะนำ**เริ่มต้นใช้งานเกตเวย์ Power BI** ขั้นตอนการปฏิบัติสั้น ๆ นี้จะช่วยให้คุณรู้ว่า เกตเวย์คืออะไร ทำงานอย่างไร และ คุณจะติดตั้ง กำหนดค่า และใช้งาน เกตเวย์ของคุณเองอย่างไร  

![](media/service-gateway-getting-started/gw_gettingstarted_0a.png)

เกตเวย์อาจเป็นเรื่องที่ต้องใช้ความรู้ทางเทคนิค และมีความซับซ้อนมาก เนื่องจากความแตกต่างของแต่ละเครือข่ายและองค์กร พักเรื่องซับซ้อนนั้นไว้ก่อน เรามาเริ่มด้วยเรื่องพื้นฐานกัน

## <a name="how-power-bi-gateways-work"></a>เกตเวย์ Power BI ทำงานอย่างไร
**เกตเวย์**เป็นซอฟต์แวร์ที่อำนวยความสะดวกในการเข้าถึงข้อมูลที่อยู่บนเครือข่ายส่วนตัวแบบติดตั้งภายในองค์กร สำหรับการใช้งานอื่นๆ ที่ตามมาในบริการระบบคลาวด์ เช่น Power BI เป็นเหมือนยามเฝ้าประตู ที่คอยฟังคำขอเชื่อมต่อ และอนุญาตเมื่อคำขอของผู้ใช้เข้าเงื่อนไขบางอย่าง (เช่น พวกเขาได้รับอนุญาตให้ใช้เกตเวย์ได้หรือไม่) ทำให้องค์กรสามารถเก็บฐานข้อมูลและคลังข้อมูลไว้บนเครือข่ายในองค์กรของพวกเขา แต่ยังให้ใช้ข้อมูลในนั้น สร้างรายงานและแดชบอร์ดที่น่าสนใจใน Power BI ได้อย่างปลอดภัย

เกตเวย์ยังรักษาความปลอดภัยของข้อมูลและการเข้าถึงข้อมูล โดยการเข้ารหัสลับ และบีบอัดข้อมูลทั้งหมดที่ส่งผ่านมัน รวมไปถึงรหัสผ่านที่ใช้เชื่อมต่อกับแหล่งข้อมูล ทั้งหมดนี้ดูตรงไปตรงมา แต่ยังมีรายละเอียดหลายเรื่องที่ต้องพิจารณา

บางครั้งคุณต้องการเกตเวย์สำหรับคุณเพียงคนเดียว – บางทีคุณมีเวิร์กบุ๊ก Excel ขนาดใหญ่ และฐานข้อมูล SQL สามตัว ที่รวบรวมข้อมูลยอดขายและการตลาดมาหลายปี และคุณต้องการสร้างแดชบอร์ด Power BI ที่แสดงยอดขายเหล่านั้นจากทุกแง่มุม มีคุณคนเดียวเท่านั้นที่สร้างรายงาน มันเป็นเวิร์กบุ๊ก Excel ของคุณ และมีเพียงคุณเท่านั้นที่ใช้ฐานข้อมูลเหล่านั้นเพื่อสร้างรายงาน Power BI คุณเพียงแค่ต้องการเกตเวย์สำหรับใช้งานส่วนบุคคล ไม่ได้แชร์แหล่งข้อมูลเหล่านั้นกับบุคคลอื่น

แต่บางที คุณอาจอยู่ในองค์กรที่มีฐานข้อมูลหลากหลายจากบริษัทต่าง ๆ กัน เช่น Analysis Services, SAP, Oracle, IBM และ แหล่งข้อมูลอื่น ๆ และคุณต้องการให้คนจำนวนมากเข้าถึงมันได้ เพื่อให้พวกเขาสามารถสร้างรายงานที่หลากหลาย ในกรณีนี้ คุณต้องการเกตเวย์ที่ช่วยให้คุณสามารถกำหนดค่าการเข้าถึงแหล่งข้อมูลเหล่านั้นทั้งหมด จากนั้นคุณก็ต้องแชร์กับบุคคลจำนวนมากในองค์กรของคุณ นั่นเป็นเกตเวยคนละประเภทกันเลย

โชคดีที่ Power BI มีเกตเวย์ให้สองแบบ ที่เข้ากับแต่ละสถานการณ์เหล่านั้นได้ดี เกตเวย์สองแบบที่ Power BI มีให้เลือกได้แก่:

* **เกตเวย์ข้อมูลในองค์กร (โหมดส่วนบุคคล)** – อนุญาตให้ผู้ใช้หนึ่งรายเชื่อมต่อกับแหล่งข้อมูล และไม่สามารถแชร์ให้กับผู้อื่นได้ สามารถใช้ได้กับ Power BI เท่านั้น
* **เกตเวย์ข้อมูลในองค์กร**– อนุญาตให้ผู้ใช้หลายคนสามารถเชื่อมต่อกับแหล่งข้อมูลในองค์กรหลายแหล่ง และสามารถใช้โดย Power BI, PowerApps, Flow และ Azure Logic ทั้งหมดด้วยการติดตั้งเกตเวย์เพียงตัวเดียว

เกตเวย์ทั้งสองแบบทำงานคล้ายกัน – พวกเขาอำนวยความสะดวกในการเข้าถึงข้อมูลที่อยู่บนเครือข่ายส่วนตัวในองค์กร เพื่อข้อมูลสามารถนำไปใช้ในบริการบนระบบคลาวด์เช่น Power BI ได้ เกตเวย์ส่วนบุคคล สามารถใช้โดยคนเพียงคนเดียว และใช้กับ Power BI เท่านั้น ในขณะที่**เกตเวย์ข้อมูลในองค์กร**สามารถใช้โดยผู้ใช้หลายคน และใช้กับหลายบริการได้

ขั้นตอนการนำเกตเวย์มาทำงาน มีอยู่ด้วยกัน สามส่วน หรือสามขั้นตอน:

* ติดตั้งเกตเวย์
* เพิ่มผู้ใช้ลงในเกตเวย์ (อนุญาตให้พวกเขาใช้เกตเวย์)
* เชื่อมต่อกับแหล่งข้อมูล

นอกจากนี้ การใช้เกตเวย์จะช่วยให้คุณทำสิ่งอื่นที่มีความสำคัญ ซึ่งก็คือ:

* รีเฟรชข้อมูลในองค์กร เพื่อให้รายงานของ Power BI ถูกปรับปรุงด้วยข้อมูลใหม่ได้

การรีเฟรชข้อมูล ช่วยให้แดชบอร์ดและรายงานของ Power BI ของคุณ มีความสดใหม่ และสะท้อนถึงข้อมูลล่าสุด ดังนั้น เมื่อมีบุคคลอื่นมาดูรายงานที่คุณสร้างขึ้นด้วยข้อมูลในองค์กร รายงานนั้นจะสามารถแสดงข้อมูลล่าสุด แม้ว่าคุณได้สร้างรายงานนั้นมาระยะนึงแล้วก็ตาม

ขั้นตอนแรก ซึ่งก็คือการติดตั้งเกตเวย์ นั้นเป็นเรื่องง่าย การอนุญาตให้ผู้ใช้สามารถเข้าถึงเกตเวย์ก็เป็นเรื่องง่ายด้วย – คุณเพียงแค่เพิ่มคนเหล่านั้นในกล่องโต้ตอบภายใน Power BI ก็เสร็จแล้ว การเชื่อมต่อกับแหล่งข้อมูลสามารถกลายเป็นเรื่องซับซ้อนขึ้นมาได้ เนื่องจากมีแหล่งข้อมูลมากมาย และแต่ละรายมีข้อกำหนดการเชื่อมต่อและรายละเอียดปลีกย่อยของตนเอง เรื่องการรีเฟรชข้อมูลเราจะอธิบายในคำแนะนำอื่น เพื่อให้เนื้อหาของบทความนี้มุ่งเน้นไปที่เกตเวย์

เรามาเริ่มด้วยสิ่งที่ง่ายเป็นอันดับแรก ดูลำดับขั้นตอนการติดตั้งเกตเวย์

## <a name="install-the-gateway"></a>ติดตั้งเกตเวย์
เมื่อต้องการติดตั้งเกตเวย์ เปิดบริการของ Power BI (คุณสามารถใช้ลิงก์นี้เพื่อเปิดใช้บริการของ Power BI ในเบราว์เซอร์ของคุณ และเข้าสู่ระบบ) และเข้าสู่ระบบ ด้วยบัญชี Power BI ของคุณ ในบริการของ Power BI เลือก**ไอคอนดาวน์โหลด**ตรงมุมบนขวา ดังที่แสดงในรูปต่อไปนี้ และเลือก**เกตเวย์ข้อมูล**

![](media/service-gateway-getting-started/gw_gettingstarted_01.png)

นั่นจะนำคุณไปยังหน้าดาวน์โหลด คลิกที่ปุ่ม**ดาวน์โหลดเกตเวย์**เพื่อเริ่มต้นการดาวน์โหลด

![](media/service-gateway-getting-started/gw_gettingstarted_02.png)

หน้าจอนี้อธิบายว่าเกตเวย์ทำงานอะไรบ้างอย่างย่อ ๆ หน้านี้ยังแสดง**คำเตือน**ที่สำคัญสองเรื่อง – เมื่อคุณติดตั้งเกตเวย์ มันทำงานบนคอมพิวเตอร์เครื่องที่คุณทำการติดตั้งจริง ๆ และถ้าคอมพิวเตอร์เครื่องนั้นถูกปิดไป เป็นเกตเวย์ก็จะถูกปิดไปด้วย (ดังนั้นระบบจะไม่ทำงาน) อีกเรื่องคือ การติดตั้งบนคอมพิวเตอร์ที่ใช้เครือข่ายไร้สายไม่ใช่สิ่งที่ดีที่สุด ดังนั้นคุณควรใช้คอมพิวเตอร์ที่เชื่อมต่อกับเครือข่ายแบบมีสาย

เมื่อคุณพร้อม เลือก**ถัดไป**เพื่อต่อไปยังการตั้งค่า

![](media/service-gateway-getting-started/gw_gettingstarted_03.png)

นี่คือที่ที่คุณตัดสินใจชนิดของเกตเวย์ที่คุณจะติดตั้ง – เกตเวย์ในองค์กร หรือเกตเวย์ส่วนบุคคล ในคำแนะนำนี้ เราจะติดตั้งการ**เกตเวย์ข้อมูลในองค์กร**

มีบางเรื่องที่ต้องรู้ตรงจุดนี้:

* เกตเวย์ทั้งสองแบบ จำเป็นต้องใช้ระบบปฏิบัติการ Windows เวอร์ชัน 64 บิต
* เกตเวย์ไม่สามารถติดตั้งบนตัวควบคุมโดเมน
* คุณสามารถติดตั้งเกตเวย์ข้อมูลในองค์กร สูงสุดสองตัวบนคอมพิวเตอร์เครื่องเดียวกัน โดยมีหนึ่งตัวต่อหนึ่งโหมด (ส่วนบุคคล และมาตรฐาน) 
* คุณไม่สามารถมีเกตเวย์มากกว่าหนึ่งตัว ที่ทำงานในโหมดเดียวกัน บนคอมพิวเตอร์เครื่องเดียวกัน
* คุณสามารถติดตั้งเกตเวย์ข้อมูลในองค์กรหลายตัว บนคอมพิวเตอร์คนละเครื่อง และจัดการเหล่าเกตเวย์เหล่านั้นทั้งหมดจาก ส่วนติดต่อเพื่อจัดการเกตเวย์ของ Power BI เดียวกันได้ (ยกเว้นกรณีเกตเวย์ส่วนบุคคล ให้ดูข้อต่อไป)
* คุณสามารถมี เกตเวย์โหมดส่วนบุคคล ตัวเดียวต่อผู้ใช้ Power BI หนึ่งคนเท่านั้น ถ้าคุณติดตั้งเกตเวย์โหมดส่วนบุคคล อีกตัวสำหรับผู้ใช้เดียวกัน ถึงแม้ว่าจะอยู่บนคอมพิวเตอร์คนละเครื่อง การติดตั้งล่าสุดจะแทนที่การติดตั้งก่อนหน้านี้

เมื่อเราเลือก**ถัดไป**การติดตั้งเกตเวย์จะเริ่มต้น คุณจำเป็นต้องระบจะติดตั้งลงในไหน โดยทั่วไป ค่าเริ่มต้น จะเป็นค่าที่ดีที่สุด

![](media/service-gateway-getting-started/gw_gettingstarted_06.png)

กระบวนการติดตั้งผ่านไปได้อย่างรวดเร็ว มีการแสดงแถบสถานะให้คุณดู

![](media/service-gateway-getting-started/gw_gettingstarted_06a.png)

การติดตั้งใกล้เสร็จแล้ว คุณจำเป็นต้องระบุบัญชีผู้ใช้ ให้กับเกตเวย์ ซึ่งควรเป็นบัญชี (ชื่อผู้ใช้และรหัสผ่าน) ที่คุณใช้เพื่อเข้าสู่ระบบ Power BI เกตเวย์จะผูกกับบัญชี Power BI ของคุณ และคุณกำหนดค่าเกตเวย์จากภายในบริการของ Power BI

![](media/service-gateway-getting-started/gw_gettingstarted_07.png)

คุณกำลังจะลงชื่อเข้าใช้ ดังที่แสดงในรูปต่อไปนี้

![](media/service-gateway-getting-started/gw_gettingstarted_08.png)

เมื่อคุณลงชื่อเข้าใช้แล้ว คุณจำเป็นต้องสร้าง**คีย์การกู้คืน** เราจะกล่าวถึงเรื่องนั้นให้ลึกขึ้นในอีกบทความ แต่ในตอนนี้ ให้ทราบว่า คุณจะต้องใช้มันเวลากู้คืน หรือย้ายเกตเวย์ของคุณ

![](media/service-gateway-getting-started/gw_gettingstarted_09.png)

เมื่อทุกอย่างผ่านไปด้วยดี คุณจะเห็นหน้าต่างที่บอกคุณว่า เกตเวย์ของคุณพร้อมแล้ว

![](media/service-gateway-getting-started/gw_gettingstarted_10.png)

แค่นั้นเองสำหรับการติดตั้งเกตเวย์ในองค์กร ตามที่ได้สัญญากันไว้ เป็นกระบวนการที่ง่าย ขั้นตอนถัดไป คือการ**เพิ่มผู้ใช้**หรือ**เพิ่มแหล่งข้อมูล** – คุณสามารถทำอย่างใดอย่างหนึ่งก่อน และทำอีกอย่างเพิ่มหลังจากคุณกำหนดค่าเริ่มต้นแล้ว

ในส่วนถัดไปจะอธิบายการเพิ่มผู้ใช้ลงในเกตเวย์ และหลังจากนั้นเราจะอธิบายว่าจะเพิ่มแหล่งข้อมูลให้กับเกตเวย์ได้ที่ไหน

## <a name="add-users-to-a-gateway"></a>เพิ่มผู้ใช้ลงในเกตเวย์
ตอนนี้เรามีเกตเวย์ที่ติดตั้งเสร็จแล้ว เราจัดการเกตเวย์จาก**บริการของ Power BI** เวลาไปยังหน้าจอการจัดการเกตเวย์ ในบริการของ Power BI ให้เลือกไอคอนรูปเฟืองที่มุมบนขวา จากนั้นเลือก**จัดการเกตเวย์**

![](media/service-gateway-getting-started/gw_gettingstarted_15.png)

จะปรากฏหน้าขึ้นภายใน บริการของ Power BI นี่เป็นที่ที่คุณสามารถจัดการเกตเวย์ของคุณ หน้า**ตั้งค่าเกตเวย์**มีลักษณะดังต่อไปนี้

![](media/service-gateway-getting-started/gw_gettingstarted_12.png)

ถ้าคุณแตะ หรือคลิกที่**ผู้ดูแล**คุณจะเห็นหน้า การจัดการผู้ดูแล ดังต่อไปนี้ โปรดทราบว่า นี่เป็นเพียงที่ที่ผู้ใช้สามารถ*ดูแล*เกตเวย์ ส่วนที่ที่จะเพิ่ม (หรือลบ) ผู้ใช้งานเกตเวย์จากแต่ละแหล่งข้อมูล จะอยู่อีกหน้าหนึ่ง – ที่เราจะไปสำรวจในอีกสองสามย่อหน้าถัดไป

![](media/service-gateway-getting-started/gw_gettingstarted_13.png)

ทันทีที่คุณติดตั้งแหล่งข้อมูล และตรวจสอบความถูกต้องของแหล่งข้อมูล (ว่าเชื่อมต่อได้สำเร็จ) มันจะแสดงใต้เกตเวย์ที่เกี่ยวข้อง ทางด้านซ้ายของหน้าจอ**จัดการเกตเวย์**นี้ ดังที่แสดงในรูปต่อไปนี้ โปรดสังเกตว่า ในบานหน้าต่างด้านขวา คุณสามารถสลับไปมาระหว่าง: **ตั้งค่าแหล่งข้อมูล**และ**ผู้ใช้** หน้าจอต่อไปนี้คือหน้าจอ**ตั้งค่าแหล่งข้อมูล**

![](media/service-gateway-getting-started/gw_gettingstarted_16.png)

เมื่อคุณเลือก**ผู้ใช้**คุณจะได้รับกล่องข้อความที่สามารถพิมพ์ผู้ใช้จากองค์กรของคุณ ที่คุณต้องการให้สิทธิ์ในการเข้าถึงแหล่งข้อมูลที่คุณเลือก ในหน้าจอต่อไปนี้ คุณจะเห็นว่า ฉันได้เพิ่ม Maggie และ Adam แล้ว

เมื่อคุณเริ่มพิมพ์อยู่อีเมลลงในกล่องข้อความ Power BI จะแสดงรายการผู้ใช้ที่มีอีเมลที่ตรงกับที่คุณกำลังพิมพ์ คุณสามารถคลิกที่ชื่อ และเพิ่มลงในรายการ

คุณยังสามารถเพิ่มอีเมลของกลุ่ม (นามแฝง) เมื่อต้องการอนุญาตการเข้าถึงข้อมูลให้กลุ่มของบุคคล ได้เช่นเดียวอีเมลรายบุคคล

![](media/service-gateway-getting-started/gw_gettingstarted_17.png)

ทันทีที่คุณกดปุ่ม**เพิ่ม** สมาชิกที่เพิ่มจะแสดงขึ้นในกล่อง และคุณสามารถเพิ่มผู้ใช้อีกถ้าคุณต้องการ การลบผู้ใช้ออกนั้นง่ายพอกัน แค่ทำเครื่องหมายตรงชื่อของพวกเขา จากนั้นกดปุ่ม**ลบออก**ด้านล่างของกล่อง

![](media/service-gateway-getting-started/gw_gettingstarted_18.png)

มีแค่นั้นจริง จำไว้ว่า คุณต้องเพิ่มผู้ใช้ไปยังแต่ละแหล่งข้อมูลที่คุณต้องการให้สิทธิ์เข้าใช้ แต่ละแหล่งข้อมูลมีรายชื่อผู้ใช้แยกจากกัน และคุณต้องเพิ่มผู้ใช้ในแต่ละแหล่งข้อมูลแยกต่างหาก

## <a name="adding-data-sources"></a>การเพิ่มแหล่งข้อมูล
แน่นอน เพื่อทำให้เกตเวย์ของคุณทำประโยชน์ให้ คุณต้องเพิ่มแหล่งข้อมูลให้มัน นี่คือจุดที่ทำให้เกิดความซับซ้อนขึ้นในเกตเวย์ Power BI – มีแหล่งข้อมูลต่าง ๆ มากมาย และแต่ละรายการมีข้อกำหนดของตนเอง (และมักจะต้องใช้ โปรแกรมควบคุม ของตนเอง)

แต่ก่อนที่เราจะส่งต่อคุณไปยังบทความอื่น ลองดูวิธีเพิ่มแหล่งข้อมูลสักหน่อย เมื่อคุณอยู่ในหน้า**จัดการเกตเวย์**ของ**บริการของ Power BI**เลือกเกตเวย์ที่คุณต้องการเพิ่มแหล่งข้อมูล และเลือก**เพิ่มแหล่งข้อมูล**ในมุมบนซ้ายของหน้า ที่อยู่ถัดจากรายการของเกตเวย์ของคุณขึ้นไปนิดเดียว

เมื่อคุณเลือกแล้ว แผง**ตั้งค่าแหล่งข้อมูล**จะปรากฏขึ้นในบานหน้าต่างด้านขวา ดังที่แสดงในรูปต่อไปนี้ ที่นั่น คุณสามารถตั้งชื่อแหล่งข้อมูลของคุณ (ใส่ลงในกล่องข้อความ**ชื่อแหล่งข้อมูล**) และเลือก**ชนิดแหล่งข้อมูล**จากกล่องรายการแบบหล่นลง

![](media/service-gateway-getting-started/gw_gettingstarted_14.png)

เอาล่ะ ตอนนี้คุณติดตั้งเกตเวย์เสร็จแล้ว และคุณพร้อมที่จะเพิ่มแหล่งข้อมูล เยี่ยม! ดูแหล่งข้อมูลในส่วนต่อไปนี้สำหรับข้อมูลเกี่ยวกับแหล่งข้อมูล รายละเอียดเพิ่มเติมเกี่ยวกับการใช้เกตเวย์ และข้อมูลอื่น ๆ ที่มีประโยชน์
