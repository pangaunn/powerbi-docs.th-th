# <a name="whitepapers-for-power-bi"></a>เอกสารทางเทคนิคสำหรับ Power BI

คุณสามารถสำรวจหัวข้อต่าง ๆ ของ Power BI ในระดับที่ลึกยิ่งกว่าด้วยเอกสารทางเทคนิค คุณสามารถค้นหารายการของเอกสารทางเทคนิคสำหรับ Power BI ได้ที่นี่

| เอกสารทางเทคนิค | คำอธิบาย |
| --- | --- |
| [Microsoft Power BI Premium](https://aka.ms/pbipremiumwhitepaper) |อธิบาย Power BI Premium ทั้งตามที่มีอยู่เมื่อเปิดใช้งาน และพัฒนาการของ Power BI Premium |
| [วางแผนการใช้งาน Power BI Enterprise](https://aka.ms/pbienterprisedeploy) |นี่คือเอกสารทางเทคนิคที่สรุปข้อควรพิจารณาสำหรับการใช้งาน Power BI ในระดับองค์กรอย่างปลอดภัยและมีประสิทธิภาพ |
| [วางแผนความจุสำหรับการวิเคราะห์แบบฝังตัวด้วย Power BI Premium](https://aka.ms/pbiewhitepaper) |เอกสารนี้ให้คำแนะนำสำหรับนักพัฒนาแอป/ISV เพื่อใช้ในการกำหนดความจุที่เหมาะสมที่สุดสำหรับธุรกิจ |
|[กระจายเนื้อหา Power BI ไปยังผู้ใช้ที่เป็นผู้เยี่ยมชมจากภายนอกโดยใช้ Azure Active Directory B2](https://aka.ms/powerbi-b2b-whitepaper)|เอกสารนี้สรุปวิธีการกระจายเนื้อหาไปยังผู้ใช้ภายนอกองค์กรโดยใช้การรวมกันของ Azure Active Directory Business-to-business (AAD B2B)|
| [แนวทางการออกแบบที่ดีที่สุดสำหรับรายงานและวิชวล](power-bi-visualization-best-practices.md) |มอบแนวทางปฏิบัติที่ดีที่สุดสำหรับการออกแบบรายงานใน Power BI |
| [การวิเคราะห์ขั้นสูงด้วย Power BI](https://info.microsoft.com/advanced-analytics-with-power-bi.html?Is=Website) |อธิบายความสามารถในการวิเคราะห์ขั้นสูงของ Power BI รวมถึงการวิเคราะห์เชิงทำนาย การแสดงวิชวลแบบกำหนดเอง R integration และนิพจน์การวิเคราะห์ข้อมูล |
| [การกรองแบบสองทิศทาง](desktop-bidirectional-filtering.md) |อธิบายการกรองไขว้แบบสองทิศทางใน Power BI Desktop (เอกสารทางเทคนิคครอบคลุมถึง SQL Server Analysis Services 2016 ทั้งสองมีลักษณะการทำงานแบบเดียวกัน) |
| [DirectQuery ใน SQL Server 2016 Analysis Services](https://blogs.msdn.microsoft.com/analysisservices/2017/04/06/directquery-in-sql-server-2016-analysis-services-whitepaper/) |DirectQuery ได้รับการออกแบบใหม่เพื่อปรับปรุงความเร็วและประสิทธิภาพการทำงานให้ดียิ้งขึ้นสำหรับ SQL Server 2016 อย่างไรก็ตาม DirectQuery ยังคงมีความซับซ้อนอย่างมากในการทำความเข้าใจและนำมาใช้ |
| [การกำกับดูแล](service-admin-governance.md) |มอบโครงสร้างสำหรับการเพิ่มผลตอบแทนจากการลงทุนที่เกี่ยวข้องกับ Power BI |
| [การรักษาความปลอดภัย](service-admin-power-bi-security.md) |แสดงคำอธิบายโดยละเอียดของการรักษาความปลอดภัยภายใน Power BI |
| [การรักษาความปลอดภัย Tabular BI Semantic Model](http://download.microsoft.com/download/D/2/0/D20E1C5F-72EA-4505-9F26-FEF9550EFD44/Securing%20the%20Tabular%20BI%20Semantic%20Model.docx) |เอกสารนี้แนะนำแบบจำลองความปลอดภัยสำหรับ tabular BI semanticและ Power BI คุณจะได้เรียนรู้วิธีการสร้างบทบาท นำความปลอดภัยแบบไดนามิกมาใช้ กำหนดการตั้งค่าการเลียนแบบ จัดการบทบาท และเลือกวิธีการเชื่อมต่อกับแบบจำลองที่ใช้ได้ในบริบทความปลอดภัยของเครือข่ายของคุณ |
| [คำแนะนำในการวางแผนความจุสำหรับเซิร์ฟเวอร์รายงาน Power BI](report-server/capacity-planning.md) |เอกสารนี้มีวัตถุประสงค์เพื่อเสนอคำแนะนำเกี่ยวกับการวางแผนความจุสำหรับเซิร์ฟเวอร์รายงาน Power BI โดยแบ่งปันผลลัพธ์การดำเนินการทดสอบการโหลดจำนวนมากของปริมาณงานต่าง ๆ กับเซิร์ฟเวอร์รายงาน |
| [การใช้งานและการวางแผน Power BI Premium](https://aka.ms/Premium-Capacity-Planning-Deployment)| เอกสารนี้มีคำแนะนำและแนวทางปฏิบัติที่ดีที่สุดสำหรับการวางแผนและการใช้งานความจุพรีเมียมสำหรับปริมาณงานที่กำหนดไว้อย่างดี|
| [Power BI และ SAP BW](https://aka.ms/powerbiandsapbw)| เอกสารนี้อธิบายถึงวิธีการที่ลูกค้า SAP สามารถใช้ประโยชน์จากการเชื่อมต่อ Power BI ไปยังระบบ SAP Business Warehouse (BW) ที่มีอยู่ของลูกค้า|
